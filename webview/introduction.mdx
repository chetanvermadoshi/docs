## Introduction

This guide explains how to implement secure webview-based authentication in web applications, specifically focusing on parent-child window communication. We support two main methods:

1. **postMessage API** - Real-time, secure cross-origin communication
2. **Query Parameters** - Simple, stateless authentication

## When to Use Each Method

<CardGroup cols={2}>
  <Card title="postMessage API" icon="message" href="/webview/postmessage">
    Best for real-time communication, sensitive data, and dynamic authentication flows
  </Card>
  <Card title="Query Parameters" icon="link" href="/webview/query-parameters">
    Best for simple, one-time authentication and easier debugging

</CardGroup>

## Authentication Data Structure

All authentication methods use the following data structure:
```typescript
interface AuthData {
  token: string;
  email: string;
  firstName: string;
  lastName: string;
  partnerUserId: string;
  pathId: string;
  segment: string;
  branchId: string;
}
```
